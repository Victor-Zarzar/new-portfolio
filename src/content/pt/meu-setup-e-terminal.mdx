## Visão Geral

Este é o meu setup completo de terminal para macOS, cobrindo desde ferramentas de shell até a configuração do prompt. Uso o **iTerm2** como emulador de terminal, **Zsh** como shell e **Starship** como prompt. Abaixo você encontra tudo o que precisa para replicar esse ambiente.

---

## Instalação

### Opção 1 — Script Automatizado (Recomendado)

Mantenho um script modular que automatiza todo o ambiente do zero. Ele instala Homebrew, ferramentas de shell, ferramentas de desenvolvimento, fontes e aplicativos de uma vez só.

```bash
git clone https://github.com/Victor-Zarzar/script-setup-macOS
cd script-setup-macOS
chmod +x setup.sh
./setup.sh
```

O script apresenta um menu interativo onde você pode executar o setup completo ou instalar componentes individualmente. Ele cuida de Homebrew, Starship, Zsh Autosuggestions, Eza, Zoxide, Bat, Zed Editor, iTerm2, NVM, Bun, Pyenv, FVM, JetBrains Mono Nerd Font e muito mais.

### Opção 2 — Instalação Manual

Se preferir instalar apenas o necessário para as configurações abaixo:

```bash
brew install starship zoxide zsh-autosuggestions eza bat nvm pyenv
brew install --cask iterm2 zed
```

## Configuração do Zsh (`~/.zshrc`)

```zsh
# Paths
eval "$(starship init zsh)"
eval "$(zoxide init zsh)"
source /opt/homebrew/share/zsh-autosuggestions/zsh-autosuggestions.zsh

export PATH=$HOME/.local/bin:$PATH

# Pyenv
export PYENV_ROOT="$HOME/.pyenv"
export PATH="$PYENV_ROOT/bin:$PATH"
eval "$(pyenv init --path)"
eval "$(pyenv init -)"

# Flutter
export PATH="$HOME/fvm/default/bin:$PATH"

# Chrome
export CHROME_EXECUTABLE="/Applications/Google Chrome.app/Contents/MacOS/Google Chrome"

# Java
export JAVA_HOME="/opt/homebrew/opt/openjdk@21"
export PATH="$JAVA_HOME/bin:$PATH"

# Bun
export BUN_INSTALL="$HOME/.bun"
export PATH="$BUN_INSTALL/bin:$PATH"

# Alias
alias cat="bat"
alias ls="eza"
alias ll="eza -l"
alias la="eza -la"
alias gs="git status"
alias ga="git add -A"
alias gl="git log"
alias gd='git diff'
alias gm='git commit -m'
alias gma='git commit -am'
alias gb='git branch'
alias gc='git checkout'

# NVM
export NVM_DIR="$HOME/.nvm"
[ -s "/opt/homebrew/opt/nvm/nvm.sh" ] && \. "/opt/homebrew/opt/nvm/nvm.sh"
[ -s "/opt/homebrew/opt/nvm/etc/bash_completion.d/nvm" ] && \. "/opt/homebrew/opt/nvm/etc/bash_completion.d/nvm"
```

## Prompt Starship (`~/.config/starship.toml`)

Crie o diretório e o arquivo de configuração:

```bash
mkdir -p ~/.config
nano ~/.config/starship.toml
```

Depois cole a configuração:

```toml

$username\
$directory\
$git_branch\
$git_status\
$docker_context\
$python\
$nodejs\
$dart\
$bun\
$package\
$cmd_duration\
$line_break\
$character"""

[username]
show_always = true
format = "🧙‍♂️ [$user]($style) "
style_user = "bold purple"

[hostname]

[directory]
```

## O que cada ferramenta faz

**Starship** — um prompt rápido e multi-shell escrito em Rust. Exibe informações contextuais como status do git, versão da linguagem atual e duração dos comandos.

**Zoxide** — um `cd` inteligente que aprende seus diretórios mais usados. Após alguns usos, `z projeto` já salta direto para a pasta certa.

**Zsh Autosuggestions** — exibe sugestões em ghost-text baseadas no seu histórico enquanto você digita. Pressione `→` para aceitar.

**Eza** — um substituto moderno do `ls` com suporte a cores, ícones e integração com git.

**Bat** — um clone do `cat` com syntax highlighting e numeração de linhas.

**iTerm2** — emulador de terminal repleto de recursos para macOS, com painéis divididos, busca e perfis configuráveis.

**Zed Editor** — editor de código acelerado por GPU, construído em Rust. Rápido, minimalista e com suporte a colaboração em tempo real.

## Zed Editor (`~/.config/zed/settings.json`)

O Zed armazena sua configuração em `~/.config/zed/settings.json`. Para abrir diretamente pelo editor, pressione `cmd-shift-p` e execute `zed: open settings`.

```json
{
  // UI
  "icon_theme": "VSCode Icons for Zed (Dark Angular)",
  "ui_font_size": 15,
  "ui_font_family": "JetBrainsMono Nerd Font",
  "buffer_font_size": 15,
  "buffer_font_family": "JetBrainsMono Nerd Font",
  "theme": {
    "mode": "dark",
    "light": "Ayu Light",
    "dark": "One Dark - Darkened"
  },
  "project_panel": {
    "auto_fold_dirs": false
  },

  // Formatação global
  "format_on_save": "on",
  "languages": {
    "Python": {
      "language_servers": [
        "ruff",
        "basedpyright",
        "!pylsp",
        "!python-refactoring"
      ],
      "formatter": {
        "language_server": { "name": "ruff" }
      },
      "code_actions_on_format": {
        "source.fixAll.ruff": true,
        "source.organizeImports.ruff": true
      }
    },
    "JavaScript": {
      "formatter": {
        "language_server": { "name": "biome" }
      },
      "code_actions_on_format": {
        "source.fixAll.biome": true,
        "source.organizeImports.biome": true
      }
    },
    "TypeScript": {
      "formatter": {
        "language_server": { "name": "biome" }
      },
      "code_actions_on_format": {
        "source.fixAll.biome": true,
        "source.organizeImports.biome": true
      }
    },
    "TSX": {
      "formatter": {
        "language_server": { "name": "biome" }
      },
      "code_actions_on_format": {
        "source.fixAll.biome": true,
        "source.organizeImports.biome": true
      }
    },
    "JSON": {
      "formatter": {
        "language_server": { "name": "biome" }
      }
    },
    "JSONC": {
      "formatter": {
        "language_server": { "name": "biome" }
      }
    },
    "CSS": {
      "formatter": {
        "language_server": { "name": "biome" }
      }
    }
  },

  // Terminal
  "terminal": {
    "shell": { "program": "/bin/zsh" },
    "env": { "TERM_PROGRAM": "iTerm.app" }
  }
}
```

**Language servers utilizados:**

- **Ruff** — linter e formatador Python extremamente rápido, escrito em Rust. Substitui o pylsp para formatação e organização de imports.
- **basedpyright** — um fork comunitário do Pyright com verificação de tipos mais rigorosa para Python.
- **Biome** — formatador e linter all-in-one para JS, TS, JSON e CSS. Substitui Prettier + ESLint em uma única ferramenta.

## Fonte

O prompt usa a **JetBrains Mono Nerd Font** para renderizar os ícones corretamente. Instale via Homebrew:

```bash
brew install --cask font-jetbrains-mono-nerd-font
```

Depois configure no iTerm2: `Preferences → Profiles → Text → Font → JetBrainsMonoNF-Regular`.
